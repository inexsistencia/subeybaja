svn up

Ej:->

svn add

=========== ADD
svn add parametro1(archivo o directorio a ser añadido)

NOTA: en el caso de ser un directorio se agregara lo que este contenga recursivamente también.
luego del add para que los cambios sean realmente agregados al repositorio deberá efectuarse un svn co (commit).

------
Ej:->
svn add lib/class.sitedomain.php
------
==============================

svn ci

=========== CI ( commit )
dir/to/commit/$ svn ci parametro1(fileToCommit) parametro2 (-m "Tiket-123 mensaje del comentario")

------
Ej:->
dev@dev-laptop61:/home/httpd/sites/www.olx.com$ svn ci lib/class.sitedomain.php -m "PROC-263 KalahariAds double redirect"
------

lo que devolvera algo como esto:

svn co

=========== CO o CHECKOUT (clona una carpeta desde el repositorio)

NOTA:antes de realizar el checkout debo situarme en la carpeta en la cual quiero se aloje lo descargado del repositorio

svn co parametro1(ruta/directorio/a/clonar/) parametro2(nombreDirectorio/)
NOTA: si no se especifica el parametro2 descargara el directorio con
el nombre que tiene en el repositoeio, y si sí se le pasa, cambiara el
nombre de el directorio por el pasado en el parametro2

------
Ej:->
svn checkout https://svn.olx.com.ar/repos/site/branches/3.23/
------
=========================

svn merge

=========== MERGE (Aplica las diferencias entre dos fuentes a una ruta de su copia local de trabajo)

NOTA:antes de realizar el merge hay que ubicarce dentro de la carpeta de el brunch correspondiente

dir/to/merge/$ svn merge parametro1(-c numerorevision) parametro2( ruta/dir/a/comparar/ )

------
Ej:->
dev@dev-laptop61:/home/httpd/sites/site/branches/3.24$ svn merge -c 61178 https://svn.olx.com.ar/repos/site/trunk
------

y esto devolvera algo como esto:
--- Merging r61178 into '.':
U lib/class.sitedomain.php

NOTA (por pablodo):svn merge copia commits de un repositorio a otro copiado.
En el caso de olx, en el sitio, por ejemplo... el codigo se encuentra en la carpeta "trunk"...
y en la carpeta branches hay copias de ese repositorio,
que vamos actualizando con los commits que queremos subir para esa version (usando svn merge)
Si hago un cambio en trunk que quiero que este en la 3.22, tendría que hacer esto:

trunk$ svn commit -m "Prueba"
Comitted to revision 1234
trunk$ cd ../branches/3.22
3.22$ svn merge -c 1234 https://svn.olx.com.ar/repos/site/trunk
3.22$ svn commit -m "Merged with revision 1234"

(pull request)

=========== (pull request)

es pedir autorización a un equipo externo,
para poder subir código a un repositorio que no es de propio.
y para ejecutarlo se debe hacer directamente desde la pagina de github.com

NOTA(por pablodo):
Acerca del pull request, es un concepto de github más que de git...
La idea del branch para svn no es la misma idea para git.
Vos en git podes hacer facilmente un branch sin tener que copiarte necesariamente un repositorio...
esta más pensado para que si vos estas laburando sobre un proyecto y necesites hacer un cambio rapido en el codigo,
no tengas que perder todo lo que hiciste, sino que haces un branch, arreglas lo que tenes que arreglar,
lo subis, y seguis laburando con el branch que estabas.
El pull request esta pensado para que vos puedas copiarte (fork) un repositorio con tu usuario,
puedas laburar sobre eso todo lo que quieras, y despues pedir que acepten tus cambios en el repositorio original.
Acá en olx se está usando para repositorios en los que quieren tener un cierto control, entonces nos dan permisos de lectura.
Nosotros nos forkeamos el proyecto con nuestro usuario, lo bajamos a nuestra maquina, laburamos,
commiteamos, pusheamos al servidor y de ahí mandamos el pull request.
